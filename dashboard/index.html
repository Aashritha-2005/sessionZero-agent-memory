SessionZero

Trust-weighted memory for AI coding agents

SessionZero is a trust-calibrated memory layer for AI coding agents, built on Cognee's real knowledge-graph lifecycle.

Pipeline · how it works
  • Ingest — commit history and decisions are structured into memory units and written to the graph via Cognee's real remember().
  • Recall — a query hits Cognee's live graph via recall(), returning ranked, semantically relevant memories.
  • Trust score — every recalled memory is scored on 4 signals before it reaches the agent.
  • Inject or flag — HIGH confidence is trusted; LOW confidence is surfaced with a warning, never silently believed.
  • Prune — stale or superseded memories are removed from the graph via real forget().
Trust signals
  • Path — how directly connected the memory is in the knowledge graph.
  • Similarity — semantic relevance to the query.
  • Recency — exponential decay from the memory's source-commit timestamp.
  • Contradiction — a newer memory explicitly supersedes this one.

SessionZero

Trust-weighted memory for AI coding agents, built on Cognee's knowledge-graph memory lifecycle. Every recalled memory is scored on path length, semantic similarity, recency, and contradiction with newer knowledge — then labeled HIGH, MEDIUM, or LOW confidence before it ever reaches the agent.

A stale or contradicted memory isn't silently dropped or silently trusted — it's surfaced with a visible warning, so the human or the agent can decide.

See the project README for the full architecture, Cognee API usage, and known limitations.
